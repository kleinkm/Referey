Referey 0.8.6

Description

Referey allows you to view your Mendeley (http://www.mendeley.com) database on your Android phone and open associated local files. Please be aware that this is not an official Mendeley application!

Referey expects the Mendeley SQLite database and the associated files on the SD card or in your Dropbox. I decided against using the Mendeley Open API, as to my knowledge part of the database is not synchronised to the Mendeley Web account and therefore not available via the API. This includes the path of associated local files.

Referey has the following functions:
- show reference list
- filter by collection or group
- filter by tags, authors, journal and publication date
- sort references
- search
- show reference details including notes and annotations (only the annotation text is shown)
- open associated files from SD card or Dropbox
- send file via email

Your use of Referey is at your sole risk. Referey is provided "as is" without expressed or implied warranty of any kind.

Since version 0.8.1, Referey opens the database with write-access for compatibility reasons. As this may change the "journal mode" of the database you should not sync the Referey database back to Mendeley Desktop.

I am aware that Referey's setup is not easy. I may add a setup wizard in the future but due other commitments this is unlikely to happen soon.

Preferences

The preference screen can be opened by pressing the menu key and choosing “Preferences”. The following options are provided:

Database path

This is the path where you have copied your SQLite database. The following page describes how you can find the Mendeley database on your computer:
http://www.mendeley.com/faq/#locate-database

The filename is usually „your_email@www.mendeley.com.sqlite“ or „online.sqlite“ if you have not entered your email in Mendeley.

PDF folder path

This is the path where you have copied the files that are associated with your references. These may include any file type. However, Referey opens these files via an Intent so you need to have an application installed that can display the associated file type.

Preserve PDF path

As the directory structure of the computer is different from an Android phone, Referey has to convert the path of associated files. „Preserve PDF path“ indicates how many levels of the saved path in the SQLite database should be preserved.

Example 1:

The associated file has the following location on your computer:
C:\Users\name\Documents\References\example.pdf

You have specified the following PDF folder path in Referey:
/sdcard/references

Preserve PDF path is set to: “file.pdf”

Alternative PDF path conversion is not checked.

Referey will then look for the file at:
/sdcard/references/example.pdf

Example 2:

The associated file has the following location on your computer:
C:\Users\name\Documents\References\Collection_A\example.pdf

You have specified the following PDF folder path in Referey:
/sdcard/references

Preserve PDF path is set to: „folder\file.pdf“

Alternative PDF path conversion is not checked.

Referey will then look for the file at:
/sdcard/references/Collection_A/example.pdf

Alternative PDF path conversion

The path conversion as described above should work fine for most users. However, if you do not rely on Mendeley's file organizer and if your PDF files are located in different folders with different folder depth it will not work. In this case, you can use Referey's alternative path conversion method.

Let's assume that you have PDF files on your computer located at
C:\Users\name\Documents\References\favourites\example1.pdf and
C:\Users\name\Documents\References\example2.pdf

You have specified the following PDF folder path in Referey:
/sdcard/references

Alternative PDF path conversion is checked.

Remove PDF path is set to: 4

Referey will then remove the first 4 folder levels (Users\name\Documents\References) from the saved path and attached the rest to the PDF folder path entered. So, Referey will look for the files at:
/sdcard/references/favourites/example1.pdf and
/sdcard/references/example2.pdf

For Windows users: The drive letter will be automatically removed and is not counted as a folder.

Font size

This defines the font size used in the main and details screen.

Use dark theme

If this is selected a black background will be used.

Main screen

The main screen shows a list of your references. References shown in blue are unread. Yellow indicates favourites and purple denotes favourite and unread (which does not make much sense but is theoretically possible).

You can select the collection or group by tapping on the left side of the action bar.

When you click on the filter symbol a dialog will appear. You can select tags, authors, the journal and the year of publication. Referey will then only show the references that fulfil all these criteria. A click on “Reset all” unselects all items. "Reset tab" only unselects the items of the current tab.

You can change the way how Referey sorts your references by clicking on the symbol on the right part of the action bar.

A long tap anywhere on the action bar will display the current filter criteria.

The details screen can be invoked by tapping on a reference.

A long tap on a reference opens the associated file.

Search

Search can be invoked by pressing the search key or by pressing the menu key and choosing “Search”. The search is performed in the selected collection or group. A search in “All documents” only searches in collections and not in groups similarly to the original Mendeley Desktop program. If several search terms are entered only references where all search terms could be found are displayed. The search is performed in title, authors, journal name, book title, abstract, year of publication, notes, annotations, tags and keywords. You can exit the search by tapping on the search bar that shows up below the action bar.

Details screen

This screen shows the details of the selected file including notes and annotations that you have made in the PDF. However, only the annotation text and not the actual position in the PDF is shown.

„Open File“ will try and open the associated file via an Intent. If more than one file is associated you can choose a file from a pop-up list. You need to have an application installed that can display this file, for example a PDF reader. This works for files that are saved locally on your computer and that have been copied to the SD card of your phone.

„Send file“ allows you to send the associated file via email.

„Cancel“ returns to the main screen.

Dropbox support

Dropbox support is experimental and requires that the original Dropbox application is installed. It does not work with current versions of the Dropbox app anymore as Dropbox has changed the undocumented feature that Referey was using. You may have success by using an older version of the Dropbox app. If the associated files are located in your Dropbox please specify the following "PDF folder path" in Preferences:
dropbox://foldername

Unfortunately, there is no documented way how Referey could inform the Dropbox application that it should refresh its file list. If Referey cannot find a file that was recently added, you may have to manually start the Dropbox application, navigate to the folder with the associated files, press the menu key and choose "Refresh".

If Referey has problems opening files from your Dropbox please enter the following information in your bug report or email: device model, Android version, Dropbox version. Otherwise I cannot try to reproduce the problem as it works fine on my phone.
